Hey, I’m building a GPT, and you’re going to help.
I’ve been working with another GPT, but that chat window is full.
We created a handover document to bring you up to speed on what we’re building and how you can continue.
When you’re ready, I’ll send that handover, followed by the Project Manager’s document with our next steps.

Core principle

This is not an MVP. We’re building a top-tier, production-grade GPT held to the highest standard of quality, stability, and polish.
Every decision, commit, and line of code should reflect that expectation.

How we work

  • One step at a time.
    No multi-step instructions. If we’re using Git Bash, you give me one command; I’ll run it and share the output; then we move to the next step.

  • Three Git Bash windows, always specified.
    We work with three windows: [SERVER], [BACKEND], and [GPT].
    Every command you give must clearly state which window it goes in (e.g., [BACKEND] git status).

  • Don’t start building yet.
    When you receive the documents, do not start writing code or making changes.
    We’ll have a brief discussion to confirm where things stand first.

  • Ask for missing context.
    If you need files or details from a previous GPT, ask me and I’ll provide them.
    If at any point you believe information from a previous GPT would help you continue effectively, let me know.
    I’ll reach out to that GPT for answers to your specific questions and bring those back to you.

  • Follow the PM plan.
    I’ll drop instructions from our Project Manager that define the next steps. Stick to them.

  • Verify every command.
    Don’t assume a command “landed.”
    After I run it, have us do a quick check so we catch issues early (e.g., expected console output, git status, health check, or a minimal log/echo).
    Build the verification into the step before moving on.

Before we begin

Do not do anything until you’ve received both documents.
After you have both, still don’t jump into work.
We’ll talk first to make sure there are no open questions for the previous GPT or the Project Manager.

Once you’ve reviewed everything, ask as many questions as you need—no limit.
The goal is absolute clarity before moving forward. Nothing should be missed, assumed, or left uncertain.

And be cognizant of our limited chat space.
Do not needlessly waste space explaining how or why every step of the way.
